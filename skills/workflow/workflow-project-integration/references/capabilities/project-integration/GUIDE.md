# 嵌入式项目集成 · 审计与指导

> 不让"看起来能跑"的项目变成"改一行崩一片"的技术债。

## 技能说明

此 Skill 以五层模型（Platform → Driver → Adapter → Middleware → APP）为基准，
对嵌入式 C 工程做**文件级别**的集成审计，并给出结构化修复清单。
也可按 8 步路线图从零指导搭建完整项目。

**参考标杆**：GR5526 SDK `graphics_lvgl_831_gpu_demo_360p` 工程 —
完整实现了 OSAL、三段式 Adapter、Manager 状态机、生产者-消费者任务、低功耗编排。

## 模式 A：项目审计工作流

### A.1 扫描工程结构

找到工程根目录（包含 `Src/`、`Keil_5/`、`CMakeLists.txt`、`Makefile` 等标志），列出完整目录树。

**若用户未指定工程路径，主动询问。**

### A.2 五层映射

将每个源文件按以下规则映射到五层模型：

| 层 | 识别标志 | 典型文件 |
|----|---------|---------|
| **Platform** | 外设初始化编排、引脚配置 | `user_periph_setup.c`、`board_init.c` |
| **Driver** | Adapter 绑定或具体硬件驱动 | `drv_adapter_port_*.c`、`*_driver.c`、`lcd_*.c` |
| **OS** | OS 抽象层、FreeRTOS 绑定 | `os_adapter/`、`osal.h`、`os_impl_*.c` |
| **Middleware** | 框架 port 层、协议栈端口 | `lv_port_*.c`、`ble_*.c` |
| **APP** | 业务逻辑、状态机、任务、UI | `main.c`、`manager/`、`task/`、`logic/`、`ui_layout/` |

### A.3 逐文件对照标准评估

按以下检查表逐文件评估，三级状态：
- ✅ **通过** — 符合标准
- ⚠️ **警告** — 部分合规，建议改进
- ❌ **缺失/违规** — 需要修复

#### Platform 层

| 检查项 | 评估标准 | 判定逻辑 |
|--------|---------|---------|
| `user_periph_setup.c` 存在 | 必须有外设初始化编排文件 | 有→✅ / 无→❌ |
| 低功耗编排位置 | `pwr_mgmt_mode_set()` 在 `app_periph_init()` 最后一行 | 在最后→✅ / 不在最后→⚠️ |
| 外设注册完整性 | 每个外设的 register 函数都在 `app_periph_init()` 中被调用 | 全部调用→✅ / 缺某个→⚠️ |
| 初始化顺序合理 | 低功耗放最后、通信总线在外设之前 | 合理→✅ |

#### Driver 层（三段式 Adapter）

对每个外设（屏幕、Flash、触摸、传感器等）单独检查：

| 检查项 | 评估标准 | 判定 |
|--------|---------|------|
| Wrapper（第一段） | 存在抽象结构体（如 `disp_drv_t`）和统一接口 | 有→✅ / 无→❌ |
| Porting（第二段） | 存在 `drv_adapter_port_*.c` 完成函数指针绑定 | 有→✅ / 无→❌ |
| Driver（第三段） | 存在具体驱动的 `.c/.h` 文件 | 有→✅ / 无→❌ |
| 三段绑定完整 | Wrapper → Porting → Driver 调用链完整 | 完整→✅ / 断开→⚠️ |
| 调用方不越层 | Middleware/APP 只调 `drv_adapter_*()`，不直接调 `lcd_*` 等 | 不越层→✅ / 直接调→❌ |

**外设检查矩阵**（审计时按实际外设展开）：

| 外设 | 一段Wrapper | 二段Porting | 三段Driver | 调用方不越层 | 综合 |
|------|:---:|:---:|:---:|:---:|:---:|
| 屏幕 | ☐ | ☐ | ☐ | ☐ | — |
| Flash | ☐ | ☐ | ☐ | ☐ | — |
| 触摸 | ☐ | ☐ | ☐ | ☐ | — |
| 传感器 | ☐ | ☐ | ☐ | ☐ | — |

#### OS 层

| 检查项 | 评估标准 | 判定 |
|--------|---------|------|
| `osal.h` 存在 | 有统一的 OSAL 头文件 | 有→✅ / 无→❌ |
| `os_impl_*.c` 存在 | 有具体 RTOS 的实现文件 | 有→✅ / 无→❌ |
| OSAL API 覆盖度 | 至少 6 个核心 API | 全有→✅ / 缺→⚠️ |
| 业务代码不越层 | `app/**/*.c` 中不出现 `xTask`/`xSemaphore`/`vTask` 等 | 无直接调用→✅ / 有→❌ |
| `FreeRTOSConfig.h` | 有且核心参数配置正确 | 有→✅ / 无→⚠️ |

**越层检查的具体做法**：在 `Src/app/` 中 grep `xTask|vTask|xSemaphore|xQueue|vPort`，命中即 ❌。

**OSAL API 覆盖度速查表**（按业务代码实际使用扫描）：

| API | 业务是否用到 | OSAL 提供 | 状态 |
|-----|:---:|:---:|:---:|
| `osal_task_create` | ☐ | ☐ | — |
| `osal_task_delay` | ☐ | ☐ | — |
| `osal_task_start` | ☐ | ☐ | — |
| `osal_sema_*` (create/take/give) | ☐ | ☐ | — |
| `osal_mutex_*` | ☐ | ☐ | — |
| `osal_queue_*` | ☐ | ☐ | — |
| `osal_heap_malloc/free` | ☐ | ☐ | — |

#### Middleware 层

| 检查项 | 评估标准 | 判定 |
|--------|---------|------|
| LVGL port 通过 Adapter | `lv_port_disp.c` 的 `flush_cb` 调 `drv_adapter_disp_*()` | 是→✅ / 直接调 lcd→❌ |
| `lv_conf.h` 存在 | LVGL 配置文件 | 有→✅ / 无→❌ |
| BLE 栈初始化 | `ble_stack_init()` 在启动链路中 | 有→✅ / 无→⚠️ |
| 中间件初始化顺序 | 在 BSP 之后、APP 任务之前 | 正确→✅ |

#### APP 层

| 检查项 | 评估标准 | 判定 |
|--------|---------|------|
| `main.c` 行数 | < 50 行 | <50→✅ / 50-100→⚠️ / >100→❌ |
| `main.c` 职责 | 只做编排，无具体硬件操作 | 只编排→✅ / 含硬件操作→❌ |
| Manager 纯 C | 不 include 任何 OS/HAL/驱动头文件 | 是→✅ / 否→❌ |
| Task 间通信 | 用信号量/队列，不直接调对方函数 | 异步→✅ / 直接调→⚠️ |
| Logic 可测试 | 只调标准 C 库（`<math.h>`、`<string.h>` 等） | 是→✅ / 否→❌ |
| 启动任务自毁 | `start_tasks()` 完成后 `osal_task_delete(NULL)` | 有→✅ / 无→⚠️ |
| UI 与业务分离 | `ui_layout/` 只调 LVGL API | 是→✅ / 混合→⚠️ |

#### 横向关注点

| 检查项 | 评估标准 | 判定 |
|--------|---------|------|
| `custom_config.h` | 有集中配置文件 | 有→✅ / 无→❌ |
| 配置中心独立性 | 不 include 任何 SDK 文件 | 是→✅ / 否→⚠️ |
| `__weak` 回调使用 | 关键系统回调通过 `__weak` 覆盖 | 有→✅ / 无→⚠️ |
| 低功耗配置 | `pwr_mgmt_mode_set()` 被调用且位置正确 | 正确→✅ / 位置错→❌ |
| 堆监控 | 有定期记录堆使用量的机制 | 有→✅ / 无→⚠️ |
| 单元测试 | `tests/` 目录存在且有测试用例 | 有→✅ / 无→⚠️ |

## 关键设计原则（两种模式共享）

### 五层依赖铁律

| 规则 | 含义 | 审计检验方法 |
|------|------|------------|
| **单向依赖** | 上层可调下层，下层不能调上层 | grep 反向 include |
| **横向隔离** | 同层模块通过接口通信，不直接耦合 | 检查同层 .c 是否互相 include |
| **跨层禁止** | APP 不能跳过 Adapter 调 Driver/OS | grep APP 层是否含 `xTask`/`lcd_*` |
| **配置只读** | 所有层读 `custom_config.h`，它不读任何层 | 检查 `custom_config.h` 的 `#include` |

### Adapter 三段式速查

| 段 | 位置 | 职责 | 换外设时 |
|----|------|------|---------|
| Wrapper | SDK/通用 | 定义抽象结构体 + 统一 API | 不改 |
| Porting | 项目 `Src/driver/` | 函数指针绑定到具体驱动 | **只改这个** |
| Driver | SDK/供应商 | 寄存器级硬件操作 | 不改 |

### `__weak` vs 函数指针（选择指南）

| 场景 | 机制 | 原因 |
|------|------|------|
| 系统回调（按键、中断） | `__weak` 覆盖 | 链接期绑定，零运行时开销 |
| 多实例外设（多块屏幕） | 函数指针表 | 支持运行期多实例注册 |
