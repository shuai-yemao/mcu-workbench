# 配置、构建与验证

## 配置

- 以目标版本的 `lv_conf.h` 为唯一配置入口，记录颜色深度、刷新周期、OS、绘制后端、Widgets、字体、图片解码器和日志开关。
- 未启用的模块不得被 APP、示例或构建系统隐式引用。
- 小 RAM 目标优先确认内存分配器、绘制缓冲、字体和图片缓存预算。

## 构建与平台

- Make/CMake/Keil 只负责纳入正确版本的 LVGL 源文件和 include 路径。
- FreeRTOS、RT-Thread、裸机或 PC 模拟器差异放在 OS Wrapper/Port，不改 LVGL 业务代码。
- 软件绘制、DMA2D、PXP、VG-Lite、NemaGFX 等后端必须与芯片能力和缓存策略对应。

## 验证清单

- 编译无 v8/v9 API 混用、未定义配置或重复符号。
- 初始化后至少完成一次显示 Flush 和一次输入 Read。
- Flush Ready、Tick、锁、DMA 完成和异常路径均有证据。
- 运行测试包含空闲界面、控件交互、页面切换、长时间刷新和内存峰值。
- 报告列出版本、配置、构建命令、测试结果、变更文件和阻塞项。
