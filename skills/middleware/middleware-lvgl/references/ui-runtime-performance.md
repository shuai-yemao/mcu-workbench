# UI 与运行时性能

## UI 边界

- APP 负责页面、业务状态、导航和用户需求；LVGL 负责控件、对象树和渲染机制。
- 统一从对象树、样式、布局、事件、动画和主题分析 UI，不把业务状态藏入设备回调。
- Widget 选择应同时记录 RAM、Flash、字体、图片和刷新区域成本。
- 字体、图片、文件系统和第三方解码器按 `lv_conf.h` 显式启用。

## 性能指标

记录以下指标，而不是只描述“显示正常”：

- 帧周期、渲染耗时、Flush 耗时和输入延迟；
- 绘制缓冲大小、峰值 RAM、堆碎片和栈水位；
- 脏区域面积、DMA/GPU 利用率、丢帧和撕裂风险；
- 日志等级、异常计数、任务占用和低功耗唤醒次数。

## 优化顺序

1. 先确认版本、颜色格式、Render Mode 和缓冲所有权。
2. 再缩小无效区域、优化 Flush/DMA、调整字体和图片缓存。
3. 最后评估 GPU/Renderer、并行绘制和任务优先级。
4. 每次优化保留前后测量，避免用主观流畅度替代证据。
