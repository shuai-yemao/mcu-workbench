# Obsidian 写入协议

## 写入前

必须展示：完整 Markdown 草稿、目标 Vault、目标相对目录、文件名、是否覆盖旧笔记和差异摘要。没有用户明确确认时，只输出草稿，不写入文件。

## 文件规则

- 默认建议目录：用户指定的 Vault 中的嵌入式项目目录；不猜测本机 Vault 路径；
- 文件名：`[芯片或模块]的[文件角色]文件架构设计思路.md`；
- 已存在文件默认创建修订草稿，不覆盖原笔记；
- WikiLink 使用 `[[笔记文件名]]`；
- 写入后重新检查代码行号、Mermaid、WikiLink 和事实标注；
- 代码发生变更时，在笔记顶部标记可能过期的范围。

## 不允许

- 静默写入 Vault；
- 修改用户已有笔记而不展示 diff；
- 把本地绝对路径写入可移植笔记；
- 把推断或 AI 解释标记为硬件实测事实。
