# 学习辅导模式

先根据用户目标选择一个模式；没有明确目标时默认进入 `tutor`，不要同时执行三种模式。

| 模式 | 触发语句 | 主要输出 | 是否逐题等待 |
| --- | --- | --- | --- |
| `tutor` | 学习、掌握、讲清楚、通过提问理解 | 逐节教学、理解判断、分级 Q&A | 是 |
| `code-audit` | 分析文件、检查架构、找遗漏 | 证据表、调用链、覆盖缺口 | 否，先给审计结果 |
| `note-refresh` | 更新、补全、同步已有笔记 | 代码/笔记差异、修订草稿 | 默认否，写入前确认 |
| `project-note` | 整理项目、产出完整笔记 | 完整 Obsidian Markdown | 关键章节逐题确认 |

## 模式切换

- 用户回答“不知道”：记录为薄弱点，给不超过 50 字的解释后继续，不重复逼问。
- 用户回答“跳过/这个会”：记录为已知或跳过，不强行展开。
- 用户要求直接出笔记：先完成最小扫描，再进入 `project-note`，不能跳过证据阶段。
- 用户只要求分析代码：使用 `code-audit`，不要自动写入 Obsidian。
- 用户中断会话：输出当前章节、已确认答案、薄弱点和下一题，供下次继续。
