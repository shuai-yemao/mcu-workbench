# 项目扫描与代码证据

## 扫描范围

读取用户指定模块的 `.c/.h`、构建文件、配置文件、启动文件、测试文件和相关 Obsidian 笔记。先限制范围，再向调用者和被调用者各扩展一层，避免无边界读取整个仓库。

## 证据表

每个结论至少记录一条：

| 字段 | 内容 |
| --- | --- |
| 文件 | 相对路径 |
| 行号 | 起止行或符号范围 |
| 符号 | 函数、结构体、宏、枚举或任务 |
| 调用者 | 谁触发 |
| 被调用者 | 下层接口或回调 |
| 状态/数据 | 输入、输出、缓存或所有权 |
| 结论类型 | 已证实、用户确认、推断、未验证 |

## 必查内容

- 入口：`main`、初始化函数、任务入口、ISR、回调和测试入口；
- API：公开函数、函数指针表、句柄、结构体和错误码；
- 配置：`#define`、默认值、编译开关、链接段和外设实例；
- 调用链：调用者 → 当前模块 → 下层接口 → 结果消费者；
- 预留接口：已声明但没有实际调用路径的函数指针、宏和配置；
- 事实差异：代码与已有笔记的遗漏、过期描述和未验证假设。

扫描结束先展示“范围、证据、缺口、风险”，再开始提问或写笔记。
