# 笔记覆盖与查漏补缺

生成草稿后逐项检查，不批量跳过：

- [ ] 每个重要 `typedef struct` 及字段；
- [ ] 每个公开 API、函数指针和回调；
- [ ] 每个配置宏、默认值、可选值和编译条件；
- [ ] 错误码、异常路径、超时和恢复策略；
- [ ] 初始化顺序、启动顺序和去初始化逆序；
- [ ] 缓冲区、句柄、队列、任务和资源所有权；
- [ ] ISR/任务/线程上下文限制；
- [ ] 与其他模块的调用关系和边界；
- [ ] 代码与旧笔记的过期项；
- [ ] 仍未验证的硬件、版本和数据手册事实。

发现遗漏时一次只展示一项：

```text
[1/N] `symbol` 在草稿中缺少 `relative/path.c:line` 的字段说明。[补充? y/n]
```

用户选择补充后更新草稿，再继续下一项；选择跳过则记录为未覆盖项。
