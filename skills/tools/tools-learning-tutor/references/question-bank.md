# 教学章节与题库

## 11 节主流程

引言和总结由 AI 自动生成；其余章节按顺序完成：

1. 一句话定义：模块负责什么；
2. 实际意义：解决什么痛点，不使用会怎样；
3. 应用场景：至少两个真实使用路径；
4. 核心机制：触发者、数据流、消费者和边界条件；
5. 关键参数：默认值、公式和选择理由；
6. 实际操作：初始化、调用、错误处理和释放顺序；
7. 常见问题：现象 → 根因 → 修复 → 验证；
8. 分级 Q&A：基础、进阶、困难；
9. 总结：是什么 → 为什么 → 怎么做 → 取舍；
10. 参考资料：视频、文档、仓库、代码/PDF；
11. 引言：根据完整笔记凝练主题。

## 理解判断

通过标准：用户能在 2–3 句话中同时说出机制和设计意图。只复述函数名、只说“为了降低耦合”或只描述现象，均不通过。

不通过时只做两轮：

1. 给不超过 100 字的核心纠正；
2. 追问一个针对缺口的确认问题；

仍不清楚就标记薄弱点，继续下一节。

## Q&A 规则

- 教学章节结束后再进入 Q&A；
- 总数 3–6 题，基础/进阶/困难各 1–2 题；
- 优先覆盖用户犹豫、答错或代码中隐含的边界；
- 逐题等待回答，记录用户原答和纠正后的理解；
- 不在问题中泄露答案。
